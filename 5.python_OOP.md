## 프로그래밍 패러다임 (Programming Paradigm)

> **패러다임** : 프로그램을 바라보고 구조화하는 **사고 방식**. 대표적으로 절차 지향과 객체 지향이 있다.

### 절차 지향과 객체 지향

| 구분 | 절차 지향 (Procedural) | 객체 지향 (Object-Oriented) |
|------|------------------------|------------------------------|
| 중심 | **함수(절차)**의 순차 실행 | **객체**들의 상호작용 |
| 데이터 | 함수와 분리 | 객체 안에 데이터+기능 묶음 |
| 비유 | 요리 레시피(순서대로) | 역할 가진 직원들의 협업 |
| 장점 | 단순·직관적 | 재사용·확장·유지보수 용이 |

```python
# 절차 지향: 데이터와 함수가 따로
name = "건석"
def greet(n):
    print(f"Hi, {n}")
greet(name)

# 객체 지향: 데이터와 기능을 객체에 묶음
class Person:
    def __init__(self, name):
        self.name = name
    def greet(self):
        print(f"Hi, {self.name}")

p = Person("건석")
p.greet()
```

### 객체와 클래스

> **클래스(class)** : 객체를 만들기 위한 **설계도(틀)**  
> **객체(object)** : 설계도로 찍어낸 **실체**

```python
# 붕어빵 틀(클래스) → 붕어빵(객체)
# 클래스 하나로 객체를 여러 개 만들 수 있다
```

| 개념 | 비유 | 설명 |
|------|------|------|
| 클래스 | 붕어빵 틀 | 속성·기능의 정의 |
| 객체(인스턴스) | 붕어빵 | 틀로 만든 실제 데이터 |

## 클래스 기초

### 클래스 (Class)

> 속성(변수)과 기능(메서드)을 묶어 정의하는 설계도. 이름은 **PascalCase**로 짓는다.

```python
class Dog:
    species = "개"          # 클래스 변수

    def __init__(self, name):   # 생성자: 객체 만들 때 자동 호출
        self.name = name        # 인스턴스 변수

    def bark(self):             # 인스턴스 메서드
        print(f"{self.name}: 멍멍")
```

### 인스턴스 (Instance)

> 클래스로 **실제 생성된 객체**. `클래스이름()`으로 만든다.

```python
my_dog = Dog("바둑이")     # 인스턴스 생성
my_dog.bark()              # 바둑이: 멍멍
print(my_dog.name)         # 바둑이   (인스턴스 변수 접근)
print(my_dog.species)      # 개       (클래스 변수 접근)
```

### 클래스와 인스턴스

> 하나의 클래스로 **여러 인스턴스**를 만들 수 있고, 각 인스턴스는 **독립된 데이터**를 가진다.

```python
dog1 = Dog("바둑이")
dog2 = Dog("초코")

dog1.bark()    # 바둑이: 멍멍
dog2.bark()    # 초코: 멍멍   (서로 다른 name)

print(type(dog1))                # <class '__main__.Dog'>
print(isinstance(dog1, Dog))     # True
```

### 클래스 구성요소

| 구성요소 | 설명 | 예시 |
|----------|------|------|
| 클래스 변수 | 모든 인스턴스가 공유 | `species = "개"` |
| 인스턴스 변수 | 인스턴스마다 고유 | `self.name` |
| 생성자 `__init__` | 객체 생성 시 자동 호출 | 초기 속성 설정 |
| 인스턴스 메서드 | 객체의 동작 | `def bark(self):` |
| `self` | 인스턴스 자기 자신 | 메서드 첫 매개변수 |

> 💡 `self`는 메서드가 호출된 **그 인스턴스**를 가리킨다. `dog1.bark()`를 호출하면 `self`에 `dog1`이 자동으로 들어간다.

### 클래스 변수와 인스턴스 변수

> **클래스 변수** : 클래스에 직접 정의, **모든 인스턴스가 공유**  
> **인스턴스 변수** : `self.변수`로 정의, **인스턴스마다 별개**

```python
class Dog:
    species = "개"            # 클래스 변수 (공유)
    count = 0

    def __init__(self, name):
        self.name = name      # 인스턴스 변수 (고유)
        Dog.count += 1        # 클래스 변수 수정은 클래스명으로

dog1 = Dog("바둑이")
dog2 = Dog("초코")

print(dog1.species, dog2.species)   # 개 개   (공유)
print(dog1.name, dog2.name)         # 바둑이 초코  (고유)
print(Dog.count)                    # 2   (만들어진 개수)
```

> ⚠️ 인스턴스로 클래스 변수에 **할당**하면(`dog1.species = "강아지"`) 공유가 깨지고 그 인스턴스만의 변수가 새로 생긴다. 클래스 변수 변경은 `Dog.species`로 해야 한다.

## 메서드

> 클래스 안에 정의된 함수. 첫 인자와 데코레이터에 따라 **세 종류**로 나뉜다.

### 인스턴스 메서드 (Instance Method)

> 첫 인자로 `self`(인스턴스)를 받는다. 가장 일반적인 메서드.

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius

    def area(self):                 # 인스턴스 메서드
        return 3.14 * self.radius ** 2

c = Circle(5)
print(c.area())   # 78.5   (self에 c가 자동 전달)
```

### 클래스 메서드 (Class Method)

> `@classmethod` 데코레이터를 붙이고, 첫 인자로 `cls`(클래스)를 받는다. **클래스 변수**를 다룰 때 쓴다.

```python
class Dog:
    count = 0

    def __init__(self, name):
        self.name = name
        Dog.count += 1

    @classmethod
    def get_count(cls):        # cls = 클래스 자신
        return cls.count

Dog("바둑이")
Dog("초코")
print(Dog.get_count())   # 2
```

### 스태틱 메서드 (Static Method)

> `@staticmethod`를 붙이고, `self`·`cls` **둘 다 받지 않는다**. 클래스와 논리적으로 관련 있지만 인스턴스/클래스 상태가 필요 없는 기능.

```python
class Calculator:
    @staticmethod
    def add(a, b):         # self, cls 없음
        return a + b

print(Calculator.add(3, 5))   # 8   (객체 안 만들고 바로 사용)
```

### 메서드 활용

```python
class BankAccount:
    bank_name = "SSAFY은행"      # 클래스 변수

    def __init__(self, owner, balance=0):
        self.owner = owner          # 인스턴스 변수
        self.balance = balance

    def deposit(self, amount):      # 인스턴스 메서드: 객체 상태 변경
        self.balance += amount
        return self.balance

    @classmethod
    def get_bank_name(cls):         # 클래스 메서드: 클래스 정보
        return cls.bank_name

    @staticmethod
    def is_valid(amount):           # 스태틱 메서드: 순수 검증 로직
        return amount > 0

acc = BankAccount("건석", 1000)
acc.deposit(500)                    # 1500
print(BankAccount.get_bank_name())  # SSAFY은행
print(BankAccount.is_valid(-100))   # False
```

### 메서드 정리

| 종류 | 데코레이터 | 첫 인자 | 접근 대상 | 호출 |
|------|-----------|---------|-----------|------|
| 인스턴스 메서드 | 없음 | `self` | 인스턴스 변수 | `obj.method()` |
| 클래스 메서드 | `@classmethod` | `cls` | 클래스 변수 | `Class.method()` |
| 스태틱 메서드 | `@staticmethod` | 없음 | (상태 접근 X) | `Class.method()` |

## 참고

### 클래스와 인스턴스 간 이름 공간 (Namespace)

> 속성을 찾을 때 **인스턴스 → 클래스** 순으로 탐색한다.

```python
class Dog:
    species = "개"        # 클래스 네임스페이스

d = Dog()
print(d.species)          # 개   (인스턴스에 없으면 클래스에서 찾음)

d.species = "강아지"       # 인스턴스 네임스페이스에 새로 생성
print(d.species)          # 강아지  (인스턴스 것 우선)
print(Dog.species)        # 개      (클래스 것은 그대로)
```

| 탐색 순서 | 위치 |
|-----------|------|
| 1 | 인스턴스 네임스페이스 |
| 2 | 클래스 네임스페이스 |

### 매직 메서드 (Magic Method)

> `__이름__` 형태의 **특수 메서드**. 특정 상황에서 파이썬이 **자동 호출**한다. (던더 메서드)

```python
class Point:
    def __init__(self, x, y):     # 생성 시 호출
        self.x = x
        self.y = y

    def __str__(self):            # print() 시 호출
        return f"({self.x}, {self.y})"

    def __add__(self, other):     # + 연산 시 호출
        return Point(self.x + other.x, self.y + other.y)

p1 = Point(1, 2)
p2 = Point(3, 4)

print(p1)          # (1, 2)        ← __str__
print(p1 + p2)     # (4, 6)        ← __add__
```

| 매직 메서드 | 호출 시점 |
|-------------|-----------|
| `__init__` | 객체 생성 |
| `__str__` | `print()`, `str()` |
| `__len__` | `len()` |
| `__eq__` | `==` 비교 |
| `__add__` | `+` 연산 |

### 데코레이터 (Decorator)

> 기존 함수/메서드를 **감싸서 기능을 추가**하는 문법. `@데코레이터` 형태로 붙인다.

```python
def my_decorator(func):
    def wrapper():
        print("함수 실행 전")
        func()
        print("함수 실행 후")
    return wrapper

@my_decorator
def hello():
    print("안녕")

hello()
# 함수 실행 전
# 안녕
# 함수 실행 후
```

> `@classmethod`, `@staticmethod`, `@property`가 대표적인 내장 데코레이터다.

```python
class Circle:
    def __init__(self, radius):
        self.radius = radius

    @property                  # 메서드를 속성처럼 사용
    def area(self):
        return 3.14 * self.radius ** 2

c = Circle(5)
print(c.area)   # 78.5   ← 괄호 없이 속성처럼 접근
```

## 상속 (Inheritance)

### 상속 기본 개념

> 기존 클래스(부모)의 속성·메서드를 **물려받아** 새 클래스(자식)를 만드는 것. 코드 재사용의 핵심.

```python
class Animal:           # 부모 클래스
    def __init__(self, name):
        self.name = name
    def eat(self):
        print(f"{self.name}가 먹는다")

class Dog(Animal):      # Animal을 상속
    def bark(self):
        print("멍멍")

d = Dog("바둑이")
d.eat()    # 바둑이가 먹는다   (부모에게서 물려받음)
d.bark()   # 멍멍             (자식 고유)
```

### 부모 클래스와 자식 클래스

| 용어 | 다른 이름 | 설명 |
|------|-----------|------|
| 부모 클래스 | 상위 클래스, 슈퍼 클래스 | 물려주는 쪽 |
| 자식 클래스 | 하위 클래스, 서브 클래스 | 물려받는 쪽 |

```python
print(issubclass(Dog, Animal))   # True  (Dog는 Animal의 자식)
print(isinstance(d, Animal))     # True  (자식 인스턴스는 부모 타입이기도 함)
```

### 메서드 오버라이딩 (Method Overriding)

> 부모에게 물려받은 메서드를 자식이 **같은 이름으로 재정의**하는 것

```python
class Animal:
    def speak(self):
        print("...")

class Dog(Animal):
    def speak(self):           # 부모 메서드 덮어쓰기
        print("멍멍")

class Cat(Animal):
    def speak(self):
        print("야옹")

Dog().speak()   # 멍멍
Cat().speak()   # 야옹
```

### 다중 상속 (Multiple Inheritance)

> 여러 부모 클래스를 동시에 상속받는 것. 탐색 순서는 **MRO(Method Resolution Order)**를 따른다.

```python
class Flyer:
    def move(self):
        print("난다")

class Swimmer:
    def swim(self):
        print("헤엄친다")

class Duck(Flyer, Swimmer):    # 둘 다 상속
    pass

d = Duck()
d.move()   # 난다
d.swim()   # 헤엄친다

print(Duck.__mro__)   # 메서드 탐색 순서 확인
```

> ⚠️ 두 부모에 같은 이름의 메서드가 있으면 **먼저 쓴 부모**(MRO 순서)의 것을 사용한다.

### super() 메서드

> 자식 클래스에서 **부모 클래스의 메서드**를 호출할 때 사용. 주로 `__init__`에서 부모 초기화에 쓴다.

```python
class Animal:
    def __init__(self, name):
        self.name = name

class Dog(Animal):
    def __init__(self, name, breed):
        super().__init__(name)    # 부모의 __init__ 호출
        self.breed = breed        # 자식 고유 속성 추가

d = Dog("바둑이", "진돗개")
print(d.name)    # 바둑이   (부모가 설정)
print(d.breed)   # 진돗개   (자식이 설정)
```

> 💡 `super()`를 쓰면 부모 클래스 이름이 바뀌어도 코드를 고칠 필요가 없어 유지보수에 유리하다.
