## 자료구조 (Data Structure)

> **자료구조** : 데이터를 효율적으로 저장·관리하기 위한 구조. 파이썬은 타입마다 그 데이터를 다루는 **전용 기능(메서드)**을 제공한다.

### 메서드

> **메서드(method)** : 특정 데이터 타입(객체)에 **속해 있는 함수**. `객체.메서드()` 형태로 호출한다.

```python
numbers = [3, 1, 2]
numbers.sort()        # 리스트 객체에 속한 sort 메서드 호출
print(numbers)        # [1, 2, 3]

text = "hello"
print(text.upper())   # HELLO  (문자열 객체의 메서드)
```

| 구분 | 형태 | 예시 |
|------|------|------|
| 함수 | `함수(객체)` | `len(numbers)` |
| 메서드 | `객체.함수()` | `numbers.sort()` |

## 공통 시퀀스 메서드

> 시퀀스 타입(`str`, `list`, `tuple`, `range`)이 **공통으로 가지는** 메서드

| 메서드 | 설명 | 예시 | 결과 |
|--------|------|------|------|
| `.index(x)` | `x`의 첫 위치 반환 | `[1, 2, 3].index(2)` | `1` |
| `.count(x)` | `x`의 개수 반환 | `[1, 1, 2].count(1)` | `2` |

```python
nums = [10, 20, 30, 20]
print(nums.index(20))   # 1   (처음 등장한 위치)
print(nums.count(20))   # 2   (등장 횟수)

s = "banana"
print(s.index("a"))     # 1
print(s.count("a"))     # 3
```

> 참고: 인덱싱·슬라이싱·`len()`·`in`·`+`·`*` 도 시퀀스 공통 동작이다. (1번 파일 참고)

## 불변 시퀀스 메서드 (문자열 전용)

> 문자열은 **변경 불가능**하므로, 모든 조작 메서드는 원본을 바꾸지 않고 **새 문자열을 반환**한다.

### 탐색 및 검증

| 메서드 | 설명 | 예시 | 결과 |
|--------|------|------|------|
| `.find(x)` | `x` 위치 반환, 없으면 `-1` | `"apple".find("p")` | `1` |
| `.index(x)` | `x` 위치 반환, 없으면 **에러** | `"apple".index("p")` | `1` |
| `.startswith(x)` | `x`로 시작하는지 | `"apple".startswith("ap")` | `True` |
| `.endswith(x)` | `x`로 끝나는지 | `"apple".endswith("le")` | `True` |
| `.count(x)` | `x`의 개수 | `"banana".count("a")` | `3` |

```python
text = "hello world"

print(text.find("o"))         # 4   (못 찾으면 -1)
print(text.find("z"))         # -1
# print(text.index("z"))      # ❌ ValueError (없으면 에러)

print(text.startswith("he"))  # True
print(text.endswith("ld"))    # True
```

> ⚠️ `.find()`와 `.index()`는 동작이 같지만, **못 찾았을 때** `find`는 `-1`, `index`는 에러를 낸다.

### 문자열 조작

| 메서드 | 설명 | 예시 | 결과 |
|--------|------|------|------|
| `.replace(a, b)` | `a`를 `b`로 교체 | `"a-b".replace("-", "_")` | `"a_b"` |
| `.strip()` | 양쪽 공백 제거 | `"  hi  ".strip()` | `"hi"` |
| `.split(sep)` | `sep` 기준 분리 → 리스트 | `"a,b".split(",")` | `["a", "b"]` |
| `.join(iter)` | `iter`를 이어붙임 | `",".join(["a", "b"])` | `"a,b"` |
| `.upper()` | 대문자로 | `"abc".upper()` | `"ABC"` |
| `.lower()` | 소문자로 | `"ABC".lower()` | `"abc"` |
| `.title()` | 단어 첫 글자 대문자 | `"hi bob".title()` | `"Hi Bob"` |
| `.capitalize()` | 첫 글자만 대문자 | `"hi bob".capitalize()` | `"Hi bob"` |

```python
text = "  Hello, World  "

print(text.strip())                # "Hello, World"
print(text.strip().replace(",", "!"))  # "Hello! World"

# split / join 은 짝꿍처럼 자주 쓰임
words = "apple banana cherry".split()  # ['apple', 'banana', 'cherry']
print("-".join(words))                 # apple-banana-cherry

s = "hello"
print(s.upper())   # HELLO
print(s)           # hello   (원본은 그대로! 불변)
```

> ⚠️ 문자열 메서드는 **원본을 바꾸지 않는다.** 결과를 쓰려면 반드시 변수에 다시 받아야 한다. (`s = s.upper()`)

## 가변 시퀀스 메서드 (리스트 전용)

> 리스트는 **변경 가능**하므로, 메서드가 **원본을 직접 수정**한다. (대부분 반환값은 `None`)

### 값 추가 및 삭제

| 메서드 | 설명 | 예시 |
|--------|------|------|
| `.append(x)` | 끝에 `x` 하나 추가 | `lst.append(4)` |
| `.extend(iter)` | 끝에 여러 개 추가 | `lst.extend([4, 5])` |
| `.insert(i, x)` | `i` 위치에 `x` 삽입 | `lst.insert(0, 9)` |
| `.remove(x)` | `x` 첫 항목 삭제 | `lst.remove(2)` |
| `.pop(i)` | `i` 항목 삭제 후 **반환** | `lst.pop()` |
| `.clear()` | 전체 삭제 | `lst.clear()` |

```python
lst = [1, 2, 3]

lst.append(4)        # [1, 2, 3, 4]
lst.extend([5, 6])   # [1, 2, 3, 4, 5, 6]
lst.insert(0, 0)     # [0, 1, 2, 3, 4, 5, 6]

lst.remove(3)        # 값 3 삭제 → [0, 1, 2, 4, 5, 6]
x = lst.pop()        # 마지막 삭제하고 반환 → x=6
y = lst.pop(0)       # 인덱스 0 삭제하고 반환 → y=0

del lst[0]           # del 문으로도 삭제 가능
lst.clear()          # []  (전부 삭제)
```

> ⚠️ `.append([4, 5])`는 리스트를 **통째로 한 요소**로 넣고, `.extend([4, 5])`는 **풀어서** 넣는다.
> ```python
> a = [1, 2]; a.append([3, 4])   # [1, 2, [3, 4]]
> b = [1, 2]; b.extend([3, 4])   # [1, 2, 3, 4]
> ```

### 정렬

| 메서드/함수 | 설명 | 원본 변경 |
|-------------|------|-----------|
| `.sort()` | 리스트 정렬 (메서드) | ✅ 원본 수정 |
| `.reverse()` | 순서 뒤집기 (메서드) | ✅ 원본 수정 |
| `sorted()` | 정렬된 **새 리스트** (내장함수) | ❌ 원본 유지 |

```python
nums = [3, 1, 2]

nums.sort()                 # 원본 변경 → [1, 2, 3]
nums.sort(reverse=True)     # 내림차순 → [3, 2, 1]
nums.reverse()              # 단순 뒤집기 → [1, 2, 3]

# sorted()는 원본을 두고 새 리스트 반환
original = [3, 1, 2]
result = sorted(original)   # result=[1, 2, 3], original=[3, 1, 2]

# key로 정렬 기준 지정
words = ["banana", "kiwi", "apple"]
words.sort(key=len)         # 길이순 → ['kiwi', 'apple', 'banana']
```

> ⚠️ `.sort()`는 리스트를 **고치고 `None`을 반환**한다. `x = nums.sort()` 하면 `x`는 `None`이 되니 주의.

## 복사 (Copy)

### 객체와 참조

> 변수는 값을 **직접 담는 게 아니라**, 값(객체)이 있는 **메모리 위치를 가리킨다(참조)**. 그래서 가변 객체를 `=`로 대입하면 **같은 객체를 공유**한다.

```python
a = [1, 2, 3]
b = a            # 참조만 복사 (같은 리스트를 가리킴)

b.append(4)
print(a)         # [1, 2, 3, 4]   ← a도 바뀜!
print(a is b)    # True           ← 같은 객체
```

> 💡 `int`, `str`, `tuple` 같은 **불변 객체**는 이런 부작용이 없다. (수정 자체가 불가능하므로)

### 얕은 복사 (Shallow Copy)

> **바깥 객체는 새로** 만들지만, 내부의 중첩 객체는 **여전히 공유**한다.

```python
a = [[1, 2], [3, 4]]
b = a[:]              # 얕은 복사 (또는 list(a), a.copy())

b.append([5, 6])
print(a)              # [[1, 2], [3, 4]]   ← 바깥 리스트는 분리됨

b[0][0] = 99
print(a)              # [[99, 2], [3, 4]]  ← 안쪽 리스트는 공유! 같이 바뀜
```

| 얕은 복사 방법 | 예시 |
|----------------|------|
| 슬라이싱 | `b = a[:]` |
| `list()` | `b = list(a)` |
| `.copy()` | `b = a.copy()` |
| `copy.copy()` | `b = copy.copy(a)` |

### 깊은 복사 (Deep Copy)

> 내부의 중첩 객체까지 **전부 새로** 복사한다. 원본과 완전히 분리된다.

```python
import copy

a = [[1, 2], [3, 4]]
b = copy.deepcopy(a)   # 깊은 복사

b[0][0] = 99
print(a)               # [[1, 2], [3, 4]]  ← 원본 영향 없음
print(b)               # [[99, 2], [3, 4]]
```

| 구분 | 바깥 객체 | 내부 중첩 객체 |
|------|-----------|----------------|
| 대입 `=` | 공유 | 공유 |
| 얕은 복사 | 분리 | **공유** |
| 깊은 복사 | 분리 | 분리 |

## 비시퀀스 데이터 구조

### 딕셔너리 (dict)

> key-value 쌍을 다루는 메서드

| 메서드 | 설명 | 예시 | 결과 |
|--------|------|------|------|
| `.keys()` | 모든 key | `d.keys()` | `dict_keys([...])` |
| `.values()` | 모든 value | `d.values()` | `dict_values([...])` |
| `.items()` | (key, value) 쌍 | `d.items()` | `dict_items([...])` |
| `.get(k)` | `k`의 값, 없으면 `None` | `d.get("x")` | 안전한 접근 |
| `.setdefault(k, v)` | 없으면 추가하고 값 반환 | `d.setdefault("x", 0)` | |
| `.update(other)` | 다른 dict로 갱신 | `d.update({"y": 2})` | |
| `.pop(k)` | `k` 삭제 후 값 반환 | `d.pop("x")` | |

```python
person = {"name": "건석", "age": 25}

print(person.get("name"))        # 건석
print(person.get("phone"))       # None   (없어도 에러 안 남)
print(person.get("phone", "없음"))  # 없음  (기본값 지정)

# 반복에 자주 쓰임
for key, value in person.items():
    print(key, value)            # name 건석 → age 25

person.update({"age": 26, "job": "dev"})  # 갱신 + 추가
```

> ⚠️ `person["phone"]`는 없는 key면 **KeyError**, `person.get("phone")`는 **None**. 안전하게 접근하려면 `.get()`을 써라.

### 세트 (set)

> 중복 없는 집합을 다루는 메서드

| 메서드 | 설명 | 예시 |
|--------|------|------|
| `.add(x)` | 요소 하나 추가 | `s.add(4)` |
| `.update(iter)` | 여러 요소 추가 | `s.update([4, 5])` |
| `.remove(x)` | 삭제 (없으면 에러) | `s.remove(2)` |
| `.discard(x)` | 삭제 (없어도 통과) | `s.discard(2)` |
| `.pop()` | 임의 요소 삭제·반환 | `s.pop()` |

```python
s = {1, 2, 3}

s.add(4)            # {1, 2, 3, 4}
s.update([5, 6])    # {1, 2, 3, 4, 5, 6}

s.remove(10)        # ❌ KeyError (없는 값)
s.discard(10)       # ✅ 에러 없이 통과

# 집합 연산 메서드 (1번 파일의 | & - 와 동일)
a = {1, 2, 3}
b = {2, 3, 4}
print(a.union(b))         # {1, 2, 3, 4}
print(a.intersection(b))  # {2, 3}
print(a.difference(b))    # {1}
```

## 참고

### List Comprehension

> 반복문을 **한 줄로** 표현해 리스트를 만드는 문법

```python
# [표현식 for 변수 in 반복가능객체 if 조건]

# 기본
squares = [x ** 2 for x in range(5)]        # [0, 1, 4, 9, 16]

# 조건 필터링
evens = [x for x in range(10) if x % 2 == 0]  # [0, 2, 4, 6, 8]

# 일반 for문과 비교
result = []
for x in range(5):
    result.append(x ** 2)   # 위 squares와 동일
```

딕셔너리·세트도 컴프리헨션이 가능하다.

```python
# 딕셔너리 컴프리헨션
{x: x ** 2 for x in range(3)}   # {0: 0, 1: 1, 2: 4}

# 세트 컴프리헨션
{x % 3 for x in range(6)}       # {0, 1, 2}
```

> ⚠️ 너무 복잡한 로직을 한 줄에 욱여넣으면 오히려 가독성이 떨어진다. 길어지면 일반 `for`문이 낫다.

### 메서드 체이닝 (Method Chaining)

> 메서드의 **반환값에 다시 메서드를 연결**해 호출하는 방식

```python
text = "  Hello World  "

result = text.strip().lower().replace(" ", "_")
print(result)   # hello_world

# 단계별로 보면:
# "  Hello World  " → strip() → "Hello World"
#                   → lower() → "hello world"
#                   → replace() → "hello_world"
```

> 문자열처럼 **새 값을 반환하는** 메서드끼리 연결할 수 있다. `.sort()`처럼 `None`을 반환하는 메서드는 체이닝 불가.

### 문자 유형 판별 메서드

> 문자열이 특정 유형인지 검사해 `True`/`False`를 반환

| 메서드 | 판별 대상 | 예시 | 결과 |
|--------|-----------|------|------|
| `.isalpha()` | 알파벳 | `"abc".isalpha()` | `True` |
| `.isdigit()` | 숫자 | `"123".isdigit()` | `True` |
| `.isalnum()` | 알파벳+숫자 | `"a1".isalnum()` | `True` |
| `.isspace()` | 공백 | `"  ".isspace()` | `True` |
| `.isupper()` | 모두 대문자 | `"ABC".isupper()` | `True` |
| `.islower()` | 모두 소문자 | `"abc".islower()` | `True` |

```python
user_input = "12345"
if user_input.isdigit():
    age = int(user_input)   # 숫자일 때만 안전하게 변환
```

### 딕셔너리 확장 : defaultdict

> 없는 key에 접근해도 **에러 대신 기본값**을 자동 생성하는 딕셔너리

```python
from collections import defaultdict

# 일반 dict는 없는 key 접근 시 KeyError
counter = defaultdict(int)   # 기본값 0인 dict
for char in "banana":
    counter[char] += 1       # 없는 key도 0에서 시작

print(dict(counter))   # {'b': 1, 'a': 3, 'n': 2}

# 그룹핑에도 유용
groups = defaultdict(list)   # 기본값 빈 리스트
groups["fruit"].append("apple")   # 알아서 [] 만들고 추가
print(dict(groups))   # {'fruit': ['apple']}
```

| 인자 | 기본값 |
|------|--------|
| `defaultdict(int)` | `0` |
| `defaultdict(list)` | `[]` |
| `defaultdict(set)` | `set()` |

### 해시 테이블 (Hash Table)

> `dict`와 `set`의 **내부 동작 원리**. key를 **해시 함수**로 변환해 저장 위치를 정하므로, 검색·삽입·삭제가 평균 **O(1)** 로 매우 빠르다.

```python
# dict 조회는 인덱스를 모르고도 즉시 찾음 (해시 덕분)
person = {"name": "건석", "age": 25}
person["name"]   # 거의 즉시 (O(1))
```

| 특징 | 설명 |
|------|------|
| 빠른 조회 | 평균 O(1) (리스트 검색 O(n)보다 빠름) |
| 해시 가능 key | **변경 불가능한 타입만** key/요소로 가능 |
| 순서 | 해시 기반이라 순서 보장 안 됨(set) |

> 그래서 `dict`의 key와 `set`의 요소에는 `list` 같은 **가변 객체를 쓸 수 없다.** (해시가 불가능하므로 `TypeError`)
> ```python
> # {[1, 2]: "x"}   ❌ TypeError: unhashable type: 'list'
> {(1, 2): "x"}     # ✅ tuple은 불변이라 가능
> ```

### 파이썬 문법 규격 (PEP 8)

> 파이썬 공식 스타일 가이드. 협업·가독성을 위한 코드 작성 관례다.

| 대상 | 규칙 | 예시 |
|------|------|------|
| 변수·함수 | snake_case | `user_name`, `get_data()` |
| 클래스 | PascalCase | `class UserAccount:` |
| 상수 | UPPER_CASE | `MAX_SIZE = 100` |
| 들여쓰기 | 공백 4칸 | (탭 대신 스페이스) |
| 연산자 주위 | 양옆 공백 1칸 | `x = a + b` |
| 쉼표 뒤 | 공백 1칸 | `func(a, b)` |

```python
# ❌ PEP 8 위반
def getUser( ):
    userName="kim"
    return userName

# ✅ PEP 8 준수
def get_user():
    user_name = "kim"
    return user_name
```

> 💡 `black`, `flake8` 같은 도구로 PEP 8 준수를 자동 검사·정리할 수 있다.
