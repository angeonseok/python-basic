## 함수 (Function)

> **함수** : 특정 작업을 수행하는 코드 묶음에 이름을 붙여, 필요할 때마다 재사용할 수 있게 만든 것

함수를 쓰면 **중복 제거**, **가독성 향상**, **유지보수 용이**라는 이점을 얻는다.

### 함수 구조

```python
def 함수이름(매개변수):
    """독스트링 (함수 설명, 선택)"""
    실행할_코드
    return 반환값
```

```python
def greet(name):
    return f"안녕, {name}!"

message = greet("건석")   # 함수 호출(call)
print(message)            # 안녕, 건석!
```

| 구성 요소 | 설명 |
|-----------|------|
| `def` | 함수 정의 키워드 |
| 함수이름 | 호출 시 사용할 이름 |
| 매개변수 | 함수가 받을 입력값 (선택) |
| 독스트링 | `"""..."""` 함수 설명 (선택) |
| `return` | 결과값 반환 (선택) |

### 함수와 반환값

> 함수는 `return`을 만나면 **값을 돌려주고 즉시 종료**된다.

```python
def add(a, b):
    return a + b
    print("실행 안됨")   # return 이후라서 도달 불가

result = add(3, 5)        # 8
```

`return`이 없거나 값 없이 `return`만 쓰면 **`None`을 반환**한다.

```python
def say_hi():
    print("hi")
    # return 없음

x = say_hi()   # hi 출력
print(x)       # None
```

여러 값을 반환하면 **튜플로 묶여서** 반환된다.

```python
def min_max(nums):
    return min(nums), max(nums)   # 튜플 반환

result = min_max([3, 1, 4])       # (1, 4)
lo, hi = min_max([3, 1, 4])       # 언패킹으로 분리
```

## 매개변수와 인자

> **매개변수(parameter)** : 함수를 **정의**할 때 받는 변수  
> **인자(argument)** : 함수를 **호출**할 때 전달하는 값

```python
def greet(name):     # name → 매개변수
    print(f"Hi, {name}")

greet("건석")        # "건석" → 인자
```

### 다양한 인자 종류

#### 위치 인자 (Positional Argument)

> 정의된 **순서대로** 값이 전달된다.

```python
def introduce(name, age):
    print(f"{name}, {age}세")

introduce("건석", 25)   # 건석, 25세
```

#### 기본 인자 값 (Default Argument)

> 인자를 생략하면 **기본값**이 사용된다.

```python
def greet(name, greeting="안녕"):
    print(f"{greeting}, {name}")

greet("건석")              # 안녕, 건석
greet("건석", "반가워")     # 반가워, 건석
```

> ⚠️ 기본값이 있는 매개변수는 **없는 매개변수보다 뒤**에 와야 한다.

```python
# def f(a=1, b):  ❌ SyntaxError
def f(a, b=1):  # ✅
    ...
```

#### 키워드 인자 (Keyword Argument)

> 매개변수 **이름을 직접 지정**해서 전달한다. 순서를 바꿔도 된다.

```python
def introduce(name, age):
    print(f"{name}, {age}세")

introduce(age=25, name="건석")   # 건석, 25세
```

> ⚠️ 위치 인자는 키워드 인자보다 **앞**에 와야 한다.

```python
# introduce(name="건석", 25)  ❌ SyntaxError
introduce("건석", age=25)      # ✅
```

#### 가변 인자 `*args`

> 개수가 **정해지지 않은 위치 인자**들을 **튜플**로 받는다.

```python
def total(*args):
    print(args)        # (1, 2, 3)
    return sum(args)

total(1, 2, 3)         # 6
```

#### 가변 키워드 인자 `**kwargs`

> 개수가 **정해지지 않은 키워드 인자**들을 **딕셔너리**로 받는다.

```python
def info(**kwargs):
    print(kwargs)      # {'name': '건석', 'age': 25}

info(name="건석", age=25)
```

#### 인자 작성 순서

> 위치 → 기본값 → `*args` → 키워드 → `**kwargs` 순서를 지켜야 한다.

```python
def func(pos, default=0, *args, **kwargs):
    ...
```

## 재귀 함수 (Recursive Function)

> **자기 자신을 호출**하는 함수. 반드시 **종료 조건(기저 조건)**이 있어야 한다.

```python
def factorial(n):
    if n <= 1:          # 기저 조건 (종료점)
        return 1
    return n * factorial(n - 1)   # 자기 자신 호출

factorial(5)            # 120  (5 * 4 * 3 * 2 * 1)
```

| 요소 | 설명 |
|------|------|
| 기저 조건 | 재귀를 멈추는 조건 (필수) |
| 재귀 호출 | 더 작은 문제로 자기 자신 호출 |

> ⚠️ 기저 조건이 없으면 **무한 재귀** → `RecursionError` 발생

## 내장 함수 (Built-in Function)

> 파이썬에 **기본 내장**되어 있어 `import` 없이 바로 쓸 수 있는 함수

| 함수 | 설명 | 예시 | 결과 |
|------|------|------|------|
| `print()` | 출력 | `print("hi")` | `hi` |
| `len()` | 길이 | `len([1, 2, 3])` | `3` |
| `type()` | 타입 확인 | `type(42)` | `<class 'int'>` |
| `sum()` | 합계 | `sum([1, 2, 3])` | `6` |
| `max()` | 최댓값 | `max([3, 1, 4])` | `4` |
| `min()` | 최솟값 | `min([3, 1, 4])` | `1` |
| `abs()` | 절댓값 | `abs(-5)` | `5` |
| `sorted()` | 정렬 | `sorted([3, 1, 2])` | `[1, 2, 3]` |
| `range()` | 정수 시퀀스 | `range(3)` | `0, 1, 2` |

### map / filter / zip

```python
# map: 모든 요소에 함수 적용
nums = [1, 2, 3]
list(map(str, nums))            # ['1', '2', '3']

# filter: 조건을 만족하는 요소만
list(filter(lambda x: x > 1, nums))   # [2, 3]

# zip: 여러 시퀀스를 짝지음
names = ["a", "b"]
ages = [25, 30]
list(zip(names, ages))          # [('a', 25), ('b', 30)]
```

## 함수와 Scope

> **Scope(범위)** : 변수에 접근할 수 있는 영역. 파이썬은 **LEGB 규칙**으로 변수를 탐색한다.

| 약자 | 범위 | 설명 |
|------|------|------|
| **L**ocal | 지역 | 함수 내부 |
| **E**nclosing | 둘러싼 함수 | 중첩 함수의 바깥 함수 |
| **G**lobal | 전역 | 모듈(파일) 최상단 |
| **B**uilt-in | 내장 | 파이썬 기본 제공 (`print` 등) |

```python
x = "global"          # 전역 변수

def func():
    x = "local"       # 지역 변수 (전역과 별개)
    print(x)          # local

func()
print(x)              # global  (전역은 영향 없음)
```

### global 키워드

> 함수 내부에서 **전역 변수를 직접 수정**할 때 사용한다.

```python
count = 0

def increment():
    global count      # 전역 count를 사용하겠다 선언
    count += 1

increment()
print(count)          # 1
```

global 없이 함수 안에서 전역 변수에 **할당**하려 하면 에러가 난다.

```python
count = 0

def bad():
    count += 1        # ❌ UnboundLocalError
    # count를 지역 변수로 인식하는데, 값이 없음
```

> ⚠️ `global`은 코드 추적을 어렵게 만든다. 가급적 **인자로 받고 `return`으로 돌려주는** 방식을 권장한다.

## 함수 스타일 가이드

### 함수 이름 작성 규칙

| 규칙 | 설명 | 예시 |
|------|------|------|
| snake_case | 소문자 + 언더스코어 | `get_user_name` |
| 동사로 시작 | 동작을 나타냄 | `calculate_total`, `is_valid` |
| 명확한 이름 | 역할이 드러나게 | `send_email` (O), `do_it` (X) |
| 축약 지양 | 알아보기 쉽게 | `count` (O), `cnt` (X) |

```python
# ✅ 좋은 예
def calculate_average(numbers):
    ...

# ❌ 나쁜 예
def calc(n):
    ...
```

### 단일 책임 원칙 (Single Responsibility Principle)

> 하나의 함수는 **하나의 일만** 해야 한다. 그래야 테스트·재사용·수정이 쉽다.

```python
# ❌ 한 함수가 여러 일을 함
def process(data):
    cleaned = [x.strip() for x in data]   # 정제
    total = sum(len(x) for x in cleaned)  # 계산
    print(total)                          # 출력
    return cleaned

# ✅ 책임별로 분리
def clean(data):
    return [x.strip() for x in data]

def count_total(data):
    return sum(len(x) for x in data)
```

## Packing & Unpacking

### Packing (패킹)

> 여러 개의 값을 **하나로 묶는** 것. `*`를 쓰면 나머지를 리스트로 묶는다.

```python
# 변수 패킹
numbers = 1, 2, 3        # 튜플로 묶임 → (1, 2, 3)

# * 패킹 (나머지 묶기)
a, *b = [1, 2, 3, 4]
print(a)   # 1
print(b)   # [2, 3, 4]

first, *middle, last = [1, 2, 3, 4, 5]
print(middle)   # [2, 3, 4]
```

함수의 `*args`, `**kwargs`도 패킹의 한 형태다.

```python
def func(*args, **kwargs):
    print(args)     # (1, 2)         위치 인자를 튜플로 패킹
    print(kwargs)   # {'a': 3}       키워드 인자를 딕셔너리로 패킹

func(1, 2, a=3)
```

### Unpacking (언패킹)

> 묶여 있는 값을 **풀어서** 개별 값으로 나누는 것. `*`(시퀀스), `**`(딕셔너리)를 쓴다.

```python
# 시퀀스 언패킹
x, y, z = (1, 2, 3)
print(x, y, z)   # 1 2 3

# * 언패킹 (함수 호출 시)
nums = [1, 2, 3]
print(*nums)     # 1 2 3   (print(1, 2, 3)과 동일)

# ** 언패킹 (딕셔너리 → 키워드 인자)
def introduce(name, age):
    print(f"{name}, {age}세")

person = {"name": "건석", "age": 25}
introduce(**person)   # 건석, 25세
```

| 기호 | 대상 | 동작 |
|------|------|------|
| `*` | 시퀀스(list, tuple 등) | 요소를 풀어서 전달 |
| `**` | 딕셔너리 | key-value를 키워드 인자로 전달 |

## 참고

### 함수와 반환

> 함수는 **값을 받아 처리하고(input)**, **결과를 돌려주는(output)** 단위다.

```python
# 입력 → 처리 → 출력 흐름
def square(x):       # 입력: x
    return x ** 2    # 처리 후 출력

square(4)            # 16
```

- `return`은 함수를 **즉시 종료**시킨다.
- 반환값이 없으면 `None`이 반환된다.
- 반환값은 변수에 저장하거나 다른 함수의 인자로 바로 쓸 수 있다.

```python
print(square(square(2)))   # 16   (중첩 사용)
```

### 람다 표현식 (Lambda Expression)

> 이름 없이 **한 줄로 정의**하는 간단한 함수. `lambda 매개변수: 반환값` 형태다.

```python
# 일반 함수
def add(a, b):
    return a + b

# 람다 표현식 (동일한 동작)
add = lambda a, b: a + b
add(3, 5)   # 8
```

주로 `map`, `filter`, `sorted` 등에 **일회성 함수**로 넘길 때 쓴다.

```python
nums = [3, 1, 4, 1, 5]

# 정렬 기준 지정
sorted(nums, key=lambda x: -x)         # [5, 4, 3, 1, 1]  (내림차순)

# map과 함께
list(map(lambda x: x * 2, nums))       # [6, 2, 8, 2, 10]

# filter와 함께
list(filter(lambda x: x > 2, nums))    # [3, 4, 5]
```

| 구분 | def 함수 | lambda |
|------|----------|--------|
| 이름 | 있음 | 없음 (익명) |
| 길이 | 여러 줄 가능 | 한 줄(표현식 하나) |
| 용도 | 재사용·복잡한 로직 | 간단한 일회성 함수 |

> ⚠️ 람다는 **표현식 하나**만 담을 수 있다. 복잡한 로직은 `def`로 작성하는 게 가독성에 좋다.
