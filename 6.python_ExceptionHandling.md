## 에러와 예외

### 디버깅 (Debugging)

> **디버깅** : 코드의 오류(버그)를 찾아 고치는 과정. 에러 메시지를 읽는 것이 첫걸음이다.

```python
# 에러 메시지는 아래에서 위로 읽는다 (가장 아래가 핵심)
# Traceback (most recent call last):
#   File "main.py", line 3, in <module>
#     print(10 / 0)
# ZeroDivisionError: division by zero
#         ↑ 에러 종류        ↑ 원인 설명
```

| 디버깅 방법 | 설명 |
|-------------|------|
| 에러 메시지 읽기 | 종류·위치·원인 파악 |
| `print()` 찍기 | 중간 값 확인 |
| 디버거 사용 | 한 줄씩 실행하며 추적 |

### 에러 (Error)

> 크게 **문법 에러**와 **예외**로 나뉜다.

#### 문법 에러 (Syntax Error)

> 코드 **문법 자체가 틀려서** 실행조차 되지 않는 오류

```python
# if True          ❌ SyntaxError (콜론 누락)
# print("hi"       ❌ SyntaxError (괄호 안 닫음)
```

#### 예외 (Exception)

> 문법은 맞지만 **실행 중에 발생**하는 오류

```python
print(10 / 0)        # ZeroDivisionError
print(undefined_var) # NameError
int("abc")           # ValueError
```

### 예외

> 자주 만나는 내장 예외 종류

| 예외 | 발생 상황 | 예시 |
|------|-----------|------|
| `ZeroDivisionError` | 0으로 나눔 | `10 / 0` |
| `NameError` | 정의 안 된 이름 사용 | `print(x)` (x 없음) |
| `TypeError` | 타입이 안 맞음 | `"1" + 1` |
| `ValueError` | 타입은 맞지만 값이 부적절 | `int("abc")` |
| `IndexError` | 인덱스 범위 초과 | `[1, 2][5]` |
| `KeyError` | 없는 딕셔너리 key | `{}["x"]` |
| `AttributeError` | 없는 속성/메서드 | `"abc".push()` |
| `FileNotFoundError` | 파일을 못 찾음 | `open("없는파일")` |

```python
# TypeError vs ValueError
"1" + 1        # TypeError: 타입이 다름 (str + int)
int("abc")     # ValueError: 타입은 str로 맞지만 숫자로 변환 불가
```

## 예외 처리 (Exception Handling)

### try & except

> 예외가 발생할 수 있는 코드를 `try`에 두고, 발생 시 처리를 `except`에 적는다. 프로그램이 **죽지 않고** 계속 실행된다.

```python
try:
    result = 10 / 0
except ZeroDivisionError:
    print("0으로 나눌 수 없습니다")   # 출력됨

print("프로그램 계속 실행")            # 죽지 않고 실행됨
```

```python
# 실용 예시: 사용자 입력 검증
try:
    age = int(input("나이: "))   # "abc" 입력 시 ValueError
    print(f"내년엔 {age + 1}세")
except ValueError:
    print("숫자를 입력하세요")
```

### 복수 예외 처리

> 여러 종류의 예외를 각각 다르게 처리할 수 있다.

```python
try:
    num = int(input("숫자: "))
    result = 10 / num
except ValueError:               # 숫자 변환 실패
    print("숫자가 아닙니다")
except ZeroDivisionError:        # 0 입력
    print("0으로 나눌 수 없습니다")

# 여러 예외를 한 번에 묶어서 처리
try:
    ...
except (ValueError, TypeError):  # 튜플로 묶음
    print("값 또는 타입 오류")
```

> ⚠️ `except`는 **위에서부터** 검사하므로, 구체적인 예외를 먼저 쓰고 포괄적인 예외(`Exception`)를 뒤에 둬야 한다.

### else & finally

> `try` 구문에 추가로 붙일 수 있는 블록

| 블록 | 실행 시점 |
|------|-----------|
| `try` | 예외가 날 수 있는 코드 |
| `except` | 예외 발생 시 |
| `else` | 예외가 **안 났을 때** |
| `finally` | 예외 여부와 **상관없이 항상** |

```python
try:
    num = int("10")
except ValueError:
    print("변환 실패")
else:
    print(f"변환 성공: {num}")   # 예외 없으면 실행 → 변환 성공: 10
finally:
    print("항상 실행됨")          # 무조건 실행 (자원 정리 등에 사용)
```

```python
# finally는 파일/연결 닫기처럼 뒷정리에 유용
try:
    f = open("data.txt")
    data = f.read()
except FileNotFoundError:
    print("파일 없음")
finally:
    f.close()   # 성공하든 실패하든 파일은 닫음
```

## 참고

### 예외 처리 주의사항

> `except`를 너무 넓게 잡으면 **진짜 버그를 숨길 수 있다.**

```python
# ❌ 나쁜 예: 모든 예외를 뭉뚱그려 무시
try:
    result = process()
except:                  # 어떤 에러든 다 잡음 (오타 버그까지 숨김)
    pass

# ✅ 좋은 예: 예상되는 예외만 구체적으로
try:
    result = process()
except ValueError:
    print("값 오류 처리")
```

> ⚠️ `except:` (맨몸)나 `except Exception: pass`는 디버깅을 어렵게 만든다. **잡을 예외를 명시**하라.

### 예외 객체 다루기

> `as` 키워드로 예외 객체를 받아 **상세 정보**를 확인할 수 있다.

```python
try:
    int("abc")
except ValueError as e:       # 예외 객체를 e로 받음
    print(f"에러 발생: {e}")   # 에러 발생: invalid literal for int()...
    print(type(e))            # <class 'ValueError'>
```

`raise`로 예외를 **직접 발생**시킬 수도 있다.

```python
def set_age(age):
    if age < 0:
        raise ValueError("나이는 음수일 수 없습니다")   # 직접 예외 발생
    return age

# set_age(-5)   # ValueError: 나이는 음수일 수 없습니다
```

### EAFP & LBYL

> 예외를 대하는 두 가지 코딩 스타일

| 스타일 | 의미 | 방식 |
|--------|------|------|
| **EAFP** | Easier to Ask Forgiveness than Permission | 일단 실행하고 예외로 처리 |
| **LBYL** | Look Before You Leap | 실행 전에 미리 조건 검사 |

```python
data = {"name": "건석"}

# LBYL: 먼저 확인하고 접근
if "age" in data:
    print(data["age"])
else:
    print("키 없음")

# EAFP: 일단 시도하고 예외 처리 (파이썬에서 선호되는 편)
try:
    print(data["age"])
except KeyError:
    print("키 없음")
```

> 💡 파이썬은 **EAFP 스타일**을 선호하는 경향이 있다. (try-except가 조건문보다 의도가 명확하고 깔끔한 경우가 많음)

### 클래스의 의미와 활용

> 사용자 정의 예외도 **클래스 상속**으로 만들 수 있다. (`Exception` 클래스를 상속)

```python
class InsufficientBalanceError(Exception):   # 사용자 정의 예외
    pass

class BankAccount:
    def __init__(self, balance):
        self.balance = balance

    def withdraw(self, amount):
        if amount > self.balance:
            raise InsufficientBalanceError("잔액 부족")
        self.balance -= amount

acc = BankAccount(1000)
try:
    acc.withdraw(2000)
except InsufficientBalanceError as e:
    print(e)   # 잔액 부족
```

> 이렇게 5번(OOP)의 상속 개념이 예외 처리와 연결된다. 모든 예외는 `Exception` 클래스를 상속한 객체다.
